# Docs

Documentation is grouped by owner:

- `project-inventory.md` for the current module, route, database, and command inventory.
- `debugging-runbook.md` for local debugging steps and common failure checks.
- `api-map.md` for the backend route map.
- `architecture/` for structure, routing, platform separation, and migration notes.
- `deployment/` for hosting and environment setup.
- `native/` for Capacitor, iOS, Android, and app-store notes.
- `desktop/` for Electron packaging and desktop runtime notes.
- `api/` for backend/API documentation.
- `cleanup/` for completed cleanup summaries and audits.
