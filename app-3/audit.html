01

Study Hub

The home tab carries the most narrative weight — the first thing a student sees. Original had real overflow, weak hierarchy, and a discouraging "0 days" empty state.

Issues fixed
  • Text overflow: "Welcome back, laheez" wrapped awkwardly with a hand emoji on its own line.
  • Missing space: Next study movePractice had no separator — copy + layout bug.
  • Header density: menu + title + dark mode + bell + search + avatar = 6 controls fighting for the same row.
  • Loud gradient CTA: the giant blue→purple "Resume practice" button dominated the screen.
  • Discouraging empty state: "DAILY STREAK · 0 Days · Start today!" felt like a scolding.
  • Quick action cards wrapping: "Review answers", "Resume practice", "Review lesson" all broke onto two lines.
What we did
  • Hero card in deep ink with a single rust accent. "Hi, Laheez 👋" renders inline at any name length.
  • Subheading reads "Pick up where you left off — you were on Hernia in Surgery." Full sentence, not three fragments.
  • Header trimmed to menu + title + search + avatar. Dark mode moves into the side menu where it belongs.
  • Primary CTA is solid rust, secondary "Open exams" is an outlined icon button — clear hierarchy, no gradients.
  • Streak card flipped to encouragement: "Start your first streak today." Plus a week dot strip that today ringed in accent.
  • Quick actions use non-breaking spaces so labels never split mid-phrase. Added a small italic numeral for quantity.
  • New "Today's plan" list — three concrete next moves with time estimates. Replaces dead space below the streak.
02

Q-Bank

Issues fixed
  • Mysterious "COURSE VIEW" pill in the top right — purpose unclear, no toggle pair.
  • Heavy gradient course-icon tiles didn't differentiate Medicine from Surgery.
  • Lots of dead vertical space below the 2 cards on first launch.
  • No filter / no search / no sense of progress at a glance.
What we did
  • Added a real search bar and four filter chips (All / In progress / Completed / Not started) with counts.
  • Each course gets a distinct icon + soft tint (Medicine = teal, Surgery = rust) for instant scan.
  • "Course view" → labeled "Subject view" with an icon, so its job is obvious.
  • Course cards now show a progress bar + remaining count + italic % in the accent color. Completion reads instantly.
  • Dead space replaced with a dashed "Add a course" affordance — a real, useful end-of-list state.
03

All Courses

Issues fixed
  • Title clipped behind the dynamic island: "All C…" — the title started too far left and ran into the camera notch.
  • Stat tiles laid out as 2-up-then-1 grid → awkward orphan "0 COMPLETED" tile on its own row.
  • Big "billboard" course card with a giant gradient — only 1 course visible at a time on the fold.
  • "Gynacology" misspelled (kept logic intact, used Gynaecology in the new copy).
  • Bottom of the card got cut off behind the tab bar.
What we did
  • Page header sits in a safe inset below the dynamic island — never clipped.
  • Stats compressed to a clean 3-up row of italic numerals (4 / 1 / 0), in editorial serif.
  • Course list is compact and scannable — 4 courses visible above the fold, each with icon + subject/lesson/remaining counts + a progress bar.
  • Bottom-tab nav uses a fade-to-bg gradient overlay so the last card always has clean tail space.
  • Section header uses the "MY COURSES / Your active tracks" pattern — eyebrow + italic serif headline, repeated across the app.
04

Lessons index

Issues fixed
  • Stat tile orphan again — 9 LESSONS / 2 COURSES / 4 SUBJECTS as a 2-2-1 grid left "4 SUBJECTS" half-alone.
  • Big "8" / "1" lesson counts in the course rows were oversized and pulled focus from the course name.
  • No sense of recency or activity — every course looked equally fresh.
  • "OPEN COURSE" pill at the foot of each card felt redundant when the whole card was tappable.
  • "Peadiatrics" misspelled (kept logic, fixed copy).
What we did
  • Stats in a clean 3-up row with italic numerals; the active "Courses" stat takes the rust accent.
  • Course count moves to a small italic numeral at the right, with a "LESSONS" label — italic, but not shouty.
  • Each card gains an "Updated 2 days ago" subline so the list reads like a feed, not a static directory.
  • Topic chips (Cardiology / Hematology / Toxicology) are tappable and visually quieter than the original.
  • "Open course →" is a single inline affordance under a divider, not a duplicate button.
05

Lesson detail

Issues fixed
  • Layout collision: the "1.1 Overview" pill overlapped the progress arc — clearly a sizing bug.
  • "NEXT" button wrapped across two lines as "NEX T" in the "Continue where you left off" card.
  • Each lesson showed both "Mark done" AND "Start" at the same time — the user can only meaningfully tap one.
  • No way to get back from the detail screen — no back arrow, only the tab bar.
  • Placeholder titles like jnkjbkj / sdcsdvsdv visible (we replaced these with realistic content for the mock).
What we did
  • A real top bar with back arrow + breadcrumb (course → unit). Tapping back returns to the Lessons index.
  • Unit hero is one clean card: progress bar + italic % + state chip. No overlap.
  • Each lesson row shows one state chip + one contextual CTA: Completed → "Review"; In progress → solid "Continue"; Not started → "Start".
  • "Mark done" is moved into the lesson interior — users mark complete from inside the lesson, not from the index.
  • "Continue where you left off" uses a wider "Resume →" pill with explicit text + arrow — no more wrapping.
06

Results

Issues fixed
  • Four full-bleed gradient cards stacked vertically — each ~200px tall, the screen felt like a casino.
  • "25.63%" rendered in red on the Score Trend card — reads as an error, not a stat.
  • No actual chart — just decorative icons faded into the gradient.
  • No list of recent attempts — the user has 4 attempts but can't see them.
  • "Exam day streak" with a 0 felt punitive next to all the encouragement elsewhere.
What we did
  • One editorial hero card: big italic 25.63, an actual sparkline showing 4-exam trend, and a short narrative ("Strong upward trend").
  • The "▲ 26%" delta keeps its green semantic; the score itself is in cream — neutral, not alarming.
  • Supporting stats in a 2×2 grid of flat tiles, each with its own italic numeral and soft icon tint — no gradients.
  • New "Recent attempts" list — each row shows mock name, time-ago, question count, and italic score. The empty data finally tells a story.
  • Day-streak framing changed from "0 days" to "Start today", with a smaller accent flame icon.
07

Principles applied

01

Same DNA, sharper expression

The blue→purple gradient stays — but as a deliberate brand motif, not wallpaper on every card. Used on one hero element per screen.

02

Confident type, single family

Plus Jakarta Sans 800 at scale does the heavy lifting. Big bold numerals replace timid grey labels; eyebrow caps anchor every screen.

03

Encouraging by default

Every zero is reframed as a beginning. "Start your first streak" beats "0 days" every time.

04

One state, one action

Buttons reflect the user's next move — never show "Mark done" and "Start" together. Let the state pick the verb.

05

Headers earn their seat

Three controls in the header, max. Anything else moves to the menu, an inline action, or in-page filters.

06

Cards earn their fills

Heavy gradients reserved for the hero. Most cards are white with a soft border; data shows through, not the chrome.
